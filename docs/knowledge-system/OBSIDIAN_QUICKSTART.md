## 单点真相范围

这页只回答：

怎样在不改 markdown 源文件的前提下，把当前 `docs/` 直接当成 Obsidian vault 打开。

## 适合什么时候读

你在这些场景里应该先看这页：

- 想用更顺手的人类阅读界面浏览 `docs/`
- 想用 backlinks、local graph、双链去看知识结构
- 想验证这套文档对“非 Codex 的阅读者”是否也足够友好

## 当前事实

- `docs/` 可以直接作为 Obsidian vault 打开
- 不需要转换格式
- markdown 文件仍然是唯一源材料
- 先补好元数据和双链，再谈更复杂的 graph 或索引层，收益最大

## 快速打开

1. 打开 Obsidian
2. 选择 `Open folder as vault`
3. 选择 `D:\workspace\rag-demo\docs`
4. 先看 `README.md` 或 `VAULT_HOME.md`

## 推荐视图

- 文件树：适合按 area 逛
- Backlinks：适合看某个主题周边都连了谁
- Local graph：适合聚焦一个概念或一条阅读链
- Search：适合搜固定契约词，例如 `/api`、`desktopApi`、`runtime.config.cjs`

## 使用习惯建议

- `archive/` 默认只当历史参考，不当主阅读入口
- 文档多起来后，优先看 local graph，不急着开 global graph
- 尽量保持核心概念命名稳定，这样 backlinks 才会越用越顺

## 对这套文档的意义

Obsidian 不是新的源文件系统，只是这套 markdown 的一个阅读壳。

也就是说：

- 写作仍在仓库 markdown 中完成
- Obsidian 负责更舒服的人类浏览
- AI 暴露层、索引层、站点层都可以继续建立在同一份 `docs/` 上

## 读法建议

如果你刚开始整理这套 vault，优先顺序建议是：

1. `README.md`
2. `WIKI_SYSTEM_SCHEMA.md`
3. `DOCUMENTATION_STANDARDS.md`
4. `KNOWLEDGE_SYSTEM_INDEX.md`
5. `AI_READING_SCOPE.md`
6. 再去看具体 area 文档
